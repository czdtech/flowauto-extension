# 方案 B：资源池 + 自由拖拽映射 (Pool & Drag UI)

## 设计理念
分出明显的“图库区”和“词库区”，通过简单的点击/拖拽来建立链接。

## 适合场景
用户手头有固定的几张素材图，要反反复复用在不同的句子里。或者是复杂的 M × N 多对多映射场景。

## UI 交互草图

```text
======================================================
[Tab: 纯文本生图]  [Tab: 参考图生图/视频]  [Tab: 历史记录]
======================================================

【1. 全局参考素材池】 (把本次要用的图全扔这里)
  [图1.jpg]  [图2.png]  [图3.webp]  [图4.jpg]  [图5.jpg]
  [ ➕ 上传更多 ]

【2. 批量提示词】(每行代表一个独立任务)
> 提示词1: A boy and a girl standing in the park
  ↳ 📎 (点击从素材池勾选参考图)：已选 [图1],[图2]

> 提示词2: Two warriors fighting
  ↳ 📎 (点击从素材池勾选参考图)：已选 [图3],[图4]

> 提示词3: A group photo of five people
  ↳ 📎 (点击从素材池勾选参考图)：已选 全部 5 张图

[ ➕ 预设提示词模板 ]
======================================================
                  [ 🚀 生成批量任务 (共 3 项) ]
```

## 方案优势
这种界面最省垂直空间。图都在顶部，下面每写一行字，就点一下别针图标 `📎`，弹出一个小气泡（或者下拉菜单）将顶部的图打个勾。这完美解决了“5图3词复杂穿插”的问题，而且交互非常现代，极大地扩展了原生产品在批量场景下的能力边界。
